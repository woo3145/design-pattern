### GoF디자인 패턴

```
디자인 패턴: 소프트웨어 디자인 과정에서 자주 발생하는 문제에 대한 전형적인 해결 방식

GoF 디자인 패턴: 소프트웨어 공학에서 가장 많이 쓰는 디자인 패턴으로 총 23개의 패턴으로 구성 됨
```

#### 범위에 따른 구분

- Class

  - 클래스와 서브클래스 간의 관련성을 다룸
  - 주로 상속을 통해 구현
  - 컴파일 타임에 정적으로 결정

- Object

  - 객체간의 관련성을 다룸
  - 런타임에 동적으로 변경됨
